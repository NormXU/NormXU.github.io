os.walk()可以得到一个三元tupple(parent, dirnames, filenames)

parent：起始路径。
Dirnames：起始路径下的文件夹。
Filenames：第三个是起始路径下的文件。

yield python的使用

#生成斐波那契数列
def fab(max): 
    n, a, b = 0, 0, 1 
    while n < max: 
        yield b 
        # print b 
        a, b = b, a + b 
        n = n + 1 
		
#在for循环中调用
for n in fab(5): 
    print n 

#也可以使用next()方法查看下进程
>>> f = fab(5) 
>>> f.next() 
1 
>>> f.next() 
1 
>>> f.next() 
2 
>>> f.next() 
3 
>>> f.next() 
5


python 解释器会将其视为一个 generator，调用 fab(5) 不会执行 fab 函数，而是返回一个 iterable 对象！在 for 循环执行时，每次循环都会执行 fab 函数内部的代码，执行到 yield b 时，fab 函数就返回一个迭代值，下次迭代时，代码从 yield b 的下一条语句继续执行， 而函数的本地变量看起来和上次中断执行前是完全一样的，于是函数继续执行，直到再次遇到 yield。

K-means clustering
Finding K centroid. SO, basically, what group a given data point belongs to is defined by which centroid it is closest to.
  • - Randomly pick K centroids
  • - Assign each data point to the centroid it is closest to
  • - Recompute the centroids based on the average position of each centroid points
  • - iterate until points stop changing assignment to centroid
Decision Tree
partution the data to minimize the entropy of the data by choosing the right attribute to decide on(ID3); Decision tree are susceptible to overfitting; 为了解决这一问题，可以construct several decision tress and let them vote on the final classification(Rando)
  • - Randomly re-sample the input data for each tree
  • - Randomize a subset of attribute each step is allowed to choose
    • 如果训练集大小为N，对于每棵树而言，随机且有放回地从训练集中的抽取N个训练样本（这种采样方式称为bootstrap sample方法），作为该树的训练集 Boosting: address mis-classified data by bootstrap sample
Basic Idea: We have multiple trees. Each that uses a random sub-sample of data that we have to train on and each tree can vote on the final result

Gini不纯度
Gini不纯度的值是：1-每个标签占总数的比例的平方和.
总的集合D被分为两个集合D1，D2，假设见面为1，不见面为0

那么D1的不纯度为1-f1^2-f0^2，总数为5，见面的占了全部，则f1=1，f0=0，结果为0

D2的不纯度为1-f1^2-f0^2，f1=0.8，f0=0.2，结果为0.32

ok，那么整个分类结果的Gini不纯度就是D1/D与0的乘积 加上 D2/D与0.32的乘积，为0.16
Gini值代表了某一个分类结果的“纯度”，我们希望结果的纯度很高，这样就不需要对这一结果进行处理了。 从以上分析可以看出，Gini值越小，纯度越高，结果越好。
